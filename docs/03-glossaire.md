# Glossaire

| Terme              | Définition                                                                                             | Première occurrence |
| ------------------ | ------------------------------------------------------------------------------------------------------ | ------------------- |
| SSO                | Dans SupEvents, permet aux étudiants de se connecter via leur compte école sans créer de compte local. | §4                  |
| OIDC               | Protocole d'authentification utilisé par le SSO de l’école pour vérifier l’identité des utilisateurs.  | §4                  |
| RBAC               | Gestion des droits basée sur les rôles (étudiant, organisateur, administrateur).                       | §4                  |
| RGPD               | Règlement encadrant la gestion des données personnelles des utilisateurs.                              | §4                  |
| PCI-DSS            | Norme de sécurité liée au paiement en ligne, respectée via Stripe.                                     | §4                  |
| Webhook            | Mécanisme permettant à Stripe d'envoyer une notification au backend après paiement.                    | §6                  |
| Broker de messages | Système (RabbitMQ) permettant de gérer les communications asynchrones entre services.                  | §6                  |
| API REST           | Interface permettant la communication entre frontend et backend via HTTP.                              | §6                  |
| CRUD               | Opérations de base : créer, lire, mettre à jour, supprimer des données.                                | §6                  |
| SLA                | Engagement de disponibilité du système pour les utilisateurs.                                          | §4                  |
| P95                | Indicateur de performance représentant le temps de réponse maximum pour 95% des requêtes.              | §4                  |
| WCAG               | Norme d’accessibilité garantissant que la plateforme est utilisable par tous.                          | §4                  |
| i18n               | Gestion de plusieurs langues dans l'application.                                                       | §5                  |
| CI/CD              | Automatisation des tests et déploiements du projet.                                                    | §6                  |
| Earlybird          | Type de billet avec tarif réduit pour les premiers inscrits.                                           | §5                  |
