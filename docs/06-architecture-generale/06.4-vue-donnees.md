# §6.4 — Vue des données

## Modèle entité-relation

Ce modèle représente les données principales de SupEvents et leurs relations. Il sert de base à la conception de la base de données relationnelle.

```mermaid
erDiagram

USER ||--o{ TICKET : "réserve"
EVENT ||--o{ TICKET : "concerne"
ORGANIZER ||--o{ EVENT : "organise"
TICKET ||--|| PAYMENT : "donne lieu à"
USER ||--o{ NOTIFICATION : "reçoit"

USER {
    UUID id PK
    VARCHAR email UK
    VARCHAR password_hash
    TIMESTAMP created_at
    TIMESTAMP updated_at
}

EVENT {
    UUID id PK
    VARCHAR title
    TEXT description
    TIMESTAMP date
    UUID organizer_id FK
    INT capacity
    TIMESTAMP created_at
}

TICKET {
    UUID id PK
    UUID user_id FK
    UUID event_id FK
    VARCHAR status
    VARCHAR qr_code
    TIMESTAMP created_at
}

PAYMENT {
    UUID id PK
    UUID ticket_id FK
    VARCHAR stripe_payment_intent_id UK
    VARCHAR status
    NUMERIC amount
    TIMESTAMP created_at
}

ORGANIZER {
    UUID id PK
    UUID user_id FK
    VARCHAR company_name
}

NOTIFICATION {
    UUID id PK
    UUID user_id FK
    VARCHAR type
    TEXT message
    BOOLEAN read
    TIMESTAMP created_at
}
```
