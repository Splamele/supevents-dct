#### Diagramme C4 Context

Ce diagramme présente une vue globale du système SupEvents, en identifiant les acteurs principaux ainsi que les systèmes externes avec lesquels il interagit. Il permet de comprendre le périmètre du système et ses interactions sans entrer dans les détails techniques internes.

```mermaid
flowchart TD
    student(["👤 Étudiant"])
    organizer(["👤 Organisateur"])
    admin(["👤 Administrateur"])

    system["SupEvents"]

    stripe["Stripe"]
    sendgrid["SendGrid"]
    sso["SSO École"]
    cdn["CDN"]

    student -- "HTTPS/REST\nConsulte et s'inscrit" --> system
    organizer -- "HTTPS/REST\nGère événements" --> system
    admin -- "HTTPS/REST\nAdmin plateforme" --> system

    system -- "OIDC/OAuth2\nAuthentification" --> sso
    system -- "HTTPS/REST\nPaiement" --> stripe
    stripe -- "Webhook HTTPS\nConfirmation" --> system

    system -- "SMTP\nEmails transactionnels" --> sendgrid
    system -- "HTTPS\nDistribution médias" --> cdn
```

#### Diagramme C4 Containers

Ce diagramme détaille les principaux conteneurs applicatifs composant SupEvents. Il met en évidence les technologies utilisées ainsi que les interactions réseau entre les différents composants.

```mermaid
flowchart TD
    frontend["Frontend SPA (React)"]
    gateway["API Gateway (Nginx)"]
    backend["Backend API (NestJS)"]

    db["PostgreSQL :5432"]
    redis["Redis :6379"]
    rabbit["RabbitMQ :5672"]
    worker["Notification Worker (Node.js)"]
    s3["S3 / MinIO"]

    frontend -- "HTTPS :443" --> gateway
    gateway -- "HTTP :3000" --> backend

    backend -- "TCP :5432" --> db
    backend -- "Redis :6379" --> redis
    backend -- "AMQP :5672" --> rabbit

    rabbit --> worker
    worker -- "SMTP" --> sendgrid["SendGrid"]

    backend -- "HTTPS" --> s3
```
