#### Diagramme de séquence — Inscription à un événement

Ce diagramme représente le parcours nominal d’un utilisateur s’inscrivant à un événement, depuis la requête initiale jusqu’à la réception de la confirmation par email.

```mermaid
sequenceDiagram
    participant U as Étudiant
    participant FE as Frontend
    participant API as Backend
    participant DB as PostgreSQL
    participant Stripe
    participant MQ as RabbitMQ
    participant Notif

    U->>FE: S'inscrire
    FE->>API: POST /tickets

    API->>DB: Vérifier places
    API->>DB: Créer ticket (pending)

    API->>Stripe: Initier paiement

    alt Paiement réussi
        Stripe-->>API: Webhook succès
        API->>DB: Ticket = confirmed
        API->>MQ: ticket.confirmed
        MQ->>Notif: Consommation
        Notif->>U: Email confirmation
    else Paiement échoué
        Stripe-->>API: Erreur
    end

```

#### Diagramme de séquence — Échec de paiement

Ce diagramme décrit le comportement du système en cas d’échec de paiement lors de l’inscription à un événement.

```mermaid
sequenceDiagram
    participant U as Étudiant
    participant API as Backend
    participant DB as PostgreSQL
    participant Stripe
    participant MQ as RabbitMQ
    participant Notif

    API->>Stripe: Initier paiement

    alt Refus carte
        Stripe-->>API: Paiement refusé
    else Timeout
        Stripe-->>API: Pas de réponse
    end

    API->>DB: Ticket = cancelled
    API->>MQ: payment.failed
    MQ->>Notif: Notification
    Notif->>U: Email échec

```
